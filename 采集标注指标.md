**标注指标**

**数据集描述：**对这个网址的简单描述

**是否能打开：**看是否可以正常打开页面，如果是404或被block等无法正常显示内容的情况，都属于无法打开，如果翻墙后可以打开，就标为可以打开，然后在是否需要翻墙中打钩。

**是否需要翻墙：**对于需要翻墙才能打开的网页，打钩。

**是否需要VIP收费：**如果是支付月度、年度会员后，可以无限下载的，打钩

**是否单个文件收费：**如果单个文件下载都需要支付一笔费用的，或者按照vip点数收费，打钩

**网站类别：**期刊/出版社/其他

**文件格式：**网页中数据的格式，如果PDF，EPUB，DOCX等

**总数量：**大概的网站中期刊或书籍的总数量，如果网页中有明确的总数量的数据，直接将该数据写入，如果是期刊的文章列表，看看列表每页数量\*总页数，如果有多个分类，需要累加，如果分类比较多，就大概估一个数即可。 

**采集难度：**该网页采集或下载数据的难度，难度的判断依据如下，如果请求参数是否加密无法分辨，就按照是否直接可下载（如果vip后直接可下载也算是直接可下载，是就无加密，不能就有加密）、账号要求、板块数量、目标数量进行判断（其他反爬措施如果标注人员无法分辨）：

|**网站参数**|**简单**|**中等**|**困难**|**备注**|
| - | - | - | - | - |
|**请求参数加密**|无加密。|基础请求参数加密（例如：无 JS 混淆、无复杂算法）。|高度复杂的请求参数加密。|包含克服反爬措施的技术支持，如 IP 控制、验证码、动态搜索策略及必要时的逆向工程。|
|**账号要求**|无需账号，或账号不会被封禁。|账号在一定次数的访问 / 操作后会被封禁。|需要付费账号，且这类账号在一定次数的访问 / 操作后也会被封禁。|如果账号相关问题仅能通过技术开发解决，无需更多付费账号，则难度可能会调整。|
|**模块 / 板块**|1 个模块。|2-3 个模块。|超过 3 个模块。|每个模块都需要独立的代码开发。如果多个模块结构相似，仅需修改参数类型，则可视为同一模块，不会升级难度。|
|**目标数据量**|少于 100 万条。|100 万至 1000 万条记录之间。|超过 1000 万条记录。|超过 1000 万条记录的量级可能涉及定制定价或更高层级的分类。（此参数主要定义 “困难” 等级或显著的数据操作场景）|
|**其他反爬措施**|基础措施，如 IP 速率限制。|中级措施，如滑动验证码、基础内容加密。|高级措施，如浏览器指纹识别、复杂的点击 / 交互验证码。|任何其他遇到的反爬技术都将被记录并评估。|
